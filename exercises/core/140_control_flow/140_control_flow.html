Narrative

Control statements in C are used to define program flow. They can execute a block of code based on certain conditions, execute it repeatedly, jump to a different block of code, and make decisions. The C language supports conditional control statements, unconditional control statements, and loops as shown in the table below.

if-else Statements

These statements make a decision as to which piece of code to execute based on a given condition. The general syntax is shown below.

if (condition 1 is true)
{
    // Execute this block of code...
    Statement 1
    Statement 2
    ...
}
else if (condition 2 is true)
{
    // Execute this block of code...
    Statement 1
    Statement 2
    ...
}
else
{
    // Execute this block of code...
    Statement 3
    Statement 4
    ...
}

A few important points about if/else statements:

  • In the above example, the else if and else parts are optional.

  • There could be any number of else if statements

  • A condition must be true (non-zero) for a block of code to be executed

  • Nested if/else statements are also possible

  • Curly braces are not mandatory if a block of code has only 1 line. However it is considered a good practice to use them.

Let’s look at an example. The following block of code asks the user to enter a number and prints whether it is positive or negative. A few points about this example code:

  • You should never write code in your main function. The main function should be used to call other functions. Text books put examples directly in main to save space but you shouldn’t do that in an actual program.

  • The scanf function is a terrible way to accept input from a human. Again, you see it used in text books to create simplified examples that save space but you shouldn’t use scanf in an actual program to get input from the user. We will show you a better way to read input from the user in an upcoming exercise dedicated solely to input/output.

  • To simplify this exercise we have also omitted all input validation. Validating user input is very important. What would happen if the user entered a number that was larger than a C integer or some other nonsense? You will also learn about input validation in an upcoming lesson.

#include <stdio.h>

int main(void)
{
    int number = 0;

    printf("\nEnter a number: ");
    scanf("%d", &number);  // User input

    if(number < 0)  // less than 0, must be negative
    {
        printf("%d is negative\n");
    }
    else  // if not negative, must be positive
    {
        printf("%d is positive\n");
    }
    return 0;  // Our program's exit status
}

Feel free to compile this code and play around with it until you are comfortable with if/else statements. If you can’t read the above code and understand what it’s doing, ask the instructor to explain it before moving on.

Switch - Case Statements

The switch statement is used to test an expression for several constant integer values. Each constant value makes a case. Switch-case is used to replace the longer if-else statements where several integers are compared. The general syntax for switch-case statements is shown below.

switch (expression)
{
    case constant_1 : // Execute this case...
        Statement 1
        break;
    case constant_2 : // Execute this case...
        Statement 2
        break;
    case constant_3 : // Execute this case...
        Statement 3
        break;
    default :  // Execute this case if no case above matched...
        Statement_4
        break;
}

A few important points about switch/case statements:

  • If the constant value is equal to that of the expression, that case block is executed.

  • The break statement is used in each 'case' so that after executing a single case, it doesn’t continue checking the other cases.

  • The break statement is an unconditional control statement that immediately breaks out of the entire switch block.

  • If none of the cases are matched, the default case gets executed.

  • The default case is optional.

  • Since an enum’s underlying value is a constant integer they are often used with switch-case statements as you will see in the next example.

Here’s an example:

int main(void)
{
    enum Day
    {
        Monday = 1,
        Tuesday,
        Wednesday,
        Thursday,
        Friday,
        Saturday,
        Sunday
    };

    int day = -1;  // Initialize with a number that will hit the default case
    printf("\nEnter the day of the week (1-7): ");  // Or present menu
    scanf("%d", &day);  // User input

    switch (day)
    {
      case Monday:
      printf("Monday\n");
      break;
      case Tuesday:
      printf("Tuesday\n");
      break;
      case Wednesday:
      printf("Wednesday\n");
      break;
      case Thursday:
      printf("Thursday\n");
      break;
      case Friday:
      printf("Friday\n");
      break;
      case Saturday:
      printf("Saturday\n");
      break;
      case Sunday:
      printf("Sunday\n");
      break;
      default:
      printf("Invalid Input\n");
      break;
    }
}

While Loop

The while loop is an entry controlled loop. That means an expression is evaluated at the entry point. If it is true (non-zero), the block of code inside the while loop is executed once. The expression is checked again. This process continues until the expression evaluates to false (zero) causing control to exit the while loop entirely and continue executing the code below the loop. The general syntax is shown below.

while (expression is true)
{
    // Execute this block of code...
    Statements
    ...
}

A few important points about while loops:

  • Care must be taken inside the block of code to ensure that at some point, the expression evaluates to false. If not, the loop could go on infinitely.

  • When while(1) is used it becomes an infinite loop because 1 always evaluated to true. Creating an infinite loop in this manner is often very handy. You would exit manually using a break statement. We discuss break statements latter in this lesson.

 

 

Do-While loop

A do-while is an exit controlled loop. That means the block of code is executed first and the while expression is evaluated at the exit point. If it is true (non-zero), the block of code is executed again. If if is false (zero), the do-while loop is exited. The general syntax is shown below.

do {
  // This block of code gets executed the first time, every time.
  Statements
  ...
} while (expression is true)

A few important points about do-while loops:

  • The do-while loop always executes its body (statements) at least once, before ever reaching the control statement.

  • A common use for do-while is user input validation. For example, the first time through the loop you ask the user a question and prompt them to enter yes or no. You always need to run that code at least once, the first time. Then at the bottom, your while statement tests to ensure they entered either yes or no. If not, the loop keeps repeating, asking the question and prompting for an appropriate answer.

For loop

The for loop is another entry controlled loop similar to the while loop. Both are equally efficient. However the for loop allows you to initialize the control variable, test the control expression, and operate on the control variable on the same line. The for loop is the most common of the loops and its general syntax is shown below.

for (initialize ctrl var; expression; operate on ctrl var)
{
    // Execute this block of code...
    Statements...
}

Here’s what the for loop looks like when implemented.

for (int i = 0; i < 3; i++)
{
    printf("%d\n", i);
}
printf("Done!");

A few important points about for loops:

  • All three sections (separated by semicolons) in the for loop parenthesis are optional. For example, the following is totally valid code that would create an infinite loop.

for (;;)  // You still need the semicolons
{
    ...  // You need to manually stop the loop here in the body using a `break` statement
}

  • There are three sections inside the for loop parenthesis separated by semicolons. The first section initializes the control variable. The second section is the control statement that controls when the loop stops. As long as the control statement is TRUE (non-zero), the loop will continue. When the control statement becomes FALSE, the loop stops. The final section is basically a counting mechanism. Each iteration of the loop, the last section will update the control variable so that eventually the condition in the second section becomes false. If the programmer makes a mistake and the control statement in the middle section never becomes false, the loop will continue forever.

  • Older compilers implementing C89/90 will not let you declare your control variable inside the for loop. Starting with C99 you can now initialize the control variable inside the for loop. For example, before C99 you had to do this:

int i;
for (i = 0; i < 3; i++)

Starting with C99 we can now do this:

for (int i = 0; i < 3; i++)

Break

We have seen the use of break statements in a switch-case block. When used, it passes the control out of the block. It works exactly the same way in loops. It terminates the loop and passes the control out of the loop.

Continue

The continue statement is used inside a loop to skip the current iteration. The loop is not terminated. Instead, control is immediately passed back to the control statement and the control statement is once again tested. Here’s an example that prints "0 1 3 4" (skipping 2);

for(int i = 0; i < 5; i++)
{
    if(i == 2)
      continue;  // I don't want to print 2...
    printf("%d ",i);
}

Goto

goto is an unconditional jump to a given LABEL. The label must be within the same scope as the goto statement. Normally this restricts your jumps to the current function. Here’s an example.

int num = -1;
while (1) // infinite loop
{
    printf("Enter a number: ");
    scanf("%d", &num);  // don't use scanf to accept user input like this

    if(num == 0)
    {
        goto LABEL1;
    }
    else
        printf("Square of %d is %d\n",num,num*num);
}

LABEL1:
    printf("Break on user request...\n");
