Narrative

In a previous lesson you learned the definitions of expression, operand, operator, and statement. In this lesson you’re going to learn about data types and we’re also going to take a closer look at the available operators.

Every application needs data. The user provides input data which are processed to produce output data. The user can supply these data in many ways. For example, data can be entered from the keyboard, read from a file, or pulled from a database just to name a few. These data can be grouped into different Data Types. The C language provides several data types. They not only hold different types of data but also have different storage representations in memory. Data Types can be used for declaring identifiers, functions, or other types. These types dictate how much memory must be reserved.

Data Types

The C data types can be grouped into 4 general categories. We will discuss each of these categories in turn.

  • Basic Types: The data types used to hold arithmetic data are known as basic types. Integers are the best known and most widely used basic data type.

  • Enum Types: Enumerated (Enum) types also hold arithmetic data. The difference is that Enums can only contain a single distinct integer value from a specified list of values.

  • Derived Types: These types are derived from the basic types to perform specific tasks. For example, an integer array is a derived data type capable of holding several integer values.

  • Void Type: Void simply means "no type."

Notes

1. long long - The C standard does not mandate the size of a long long in bits, but required values of LLONG_MIN and LLONG_MAX which are listed in the limits.h header file.
2. size_t - Is used to represent the size objects in memory (not to be confused with 'object oriented programming' objects). It can represent the maximum theoretical size of an object in memory and therefore can never be negative. It’s often a typedef of unsigned long but that is not guaranteed. The C standard only says it must be at least 16 bits.
3. ssize_t - Isn’t even defined in the C standard. It’s part of the POSIX standard (Operating system standard). The ssize_t type is a signed version of size_t.
Never say always

For some types, the size may be different for different compilers. For example, an int occupies 4 bytes when compiled with GCC (64 bit system) but only 2 bytes when compiled with a Windows-based Turbo C compiler. Similarly the size of a long double could be 12 or 16 bytes depending upon the efficiency of the processor.

The sizes listed in the table above are the sizes for GCC when compiling on a 64 bit computer. Unless stated otherwise, for the duration of this course, always assume the exercises and questions are referring to GCC compiled code on a 64-bit machine.

The sizeof operator - The standard library includes an operator called sizeof for determining the size of a type. Note that sizeof is an operator, not a function. We discuss the C operators in detail later in this lesson. The sizeof operator is often confused for a function because it’s common to supply its single operand using parenthesis like this; sizeof(x); You can also code it like sizeof x and it will work just fine. See the the accepted answer to this stackoverflow question, "Why is sizeof considered an operator?"" for more detail.

Example Program - Display the size of a signed integer.

#include<stdio.h>

int main (void)
{
  size_t size;  // variable declaration
  size = sizeof(signed int); // get the size in bytes
  printf("A signed int occupies %zu bytes\n\n", size);
}

The above program produces the following output:

 A signed int occupies 4 bytes

Enum Types

Enumerated types allow us to create our own symbolic names for a list of related items. The key word for an enumeration is enum. For example, we could create an enumerated type for the primary colors as shown below.

enum primary_color
{
  red,
  green,
  blue
};  // Don't forget this semicolon

Since enum is a type, you should define it in a header (.h) file. These enumerated types can be used like any other type in a program. The type primary_color can now be used to define variables or the return type of a function. The actual enumeration can be used directly in code. For example:

int main(void)
{
    enum primary_color your_favorite_color = blue;

    if ( your_favorite_color == blue )
        printf("True blue!\n");
    else if ( your_favorite_color == red )
        printf("Russian conspiracy!\n");
    else if ( your_favorite_color == green )
        printf("Going green!\n");

    return 0;
}

When C was first introduced it did not contain a Boolean type. A Boolean (bool) is simply true or false. Programmers used to define their own bool type using enum. This was such a common use case for enum that the bool data type was eventually added to the standard (C99). The Boolean type is still just an enum but now all you need to do is include the stdbool.h header file.

Void

As we have seen, void type specifies "no type." Consequentially, variables cannot be defined with a type of void. Void can be used in the following situations:

  • Function return type: If a function doesn’t return any value, the return type is void. Although you sometimes see the main function defined with a void return type (mostly in text books, not actual practice), most modern compilers will issue a warning if main’s return type is not an integer. This is because the overall program should return a 0 if it terminated normally and a non-zero if it did not. Many applications (and the kernel) often look for this exit status.

  • Function Arguments: If a function accepts no parameters, we can (and should) specify it as void. Again, many modern compilers will issue a warning in cases where main accepts no arguments and the parenthesis are left empty like this; int main().

  • Void Pointers: A void pointer can provide the address of an object without specifying its type. Type casting has to be done to determine the type. The dynamic memory allocation functions malloc and calloc return void pointers.

Operators

Operators are the symbols used to perform specific operations. The C language provides several built-in operators that perform numerous tasks. C operators can either be classified based on the number of operands or the type of operation they perform. Based on the number of operands, C operators can either be unary, binary, or ternary. In this lesson we will discuss the operators based on the type of operation they perform. Based on the type of operation they perform, C operators are classified into the following groups that will each be discussed in turn.

  • Arithmetic Operators

  • Relational Operators

  • Bitwise Operators

  • Assignment Operators

Arithmetic Operators

While most of the operators in the following table are self explanatory, I would like to highlight the increment and decrement operators. They are the ++ and -- operators which increase/decrease the value of a variable by 1. Seems simple enough but there’s a subtle difference depending on weather or not the operator is used before or after the operand as shown below.

a++ a-- Post-increment or decrement the value.
++a --a Pre-increment or decrement the value.

The post-increment/decrement (a++) is the most common. With post-increment we use the value first and then increment/decrement it. The opposite is true for pre-increment/decrement. With pre-increment/decrement we first increase/decrease the value and then use the resulting value. This is a VERY common source of what are commonly referred to as 'one-off' errors. It’s very important that you take the time to understand the difference.

Assignment Operators

Assignment operators are normally used to assign values on the right side to a variable on the left. Arithmetic and bitwise operators can be combined with assignment operators to form several operators as shown in the following table.

Table 5. Assignment Operators

Operator

Description

=

Basic assignment operator. Assigns right side value to left side identifier.

+=  -=  *=  /=  %= 

Starting with the value of the variable on the left, apply the mathematical operator using the right operand (value), then reassign the result to the left operand.
Example: a += 6; is equivalent to a = a + 6;

&=  |=  ^=  <⇐  >>= 

Starting with the value of the variable on the left, apply the bitwise operator using the right operand (value), then reassign the result to the left operand.
Example: a ^= 6; is equivalent to a = a ^ 6;

Example: <expression> ? 1 : 0

In other words if the expression on the left is true, do what comes after the question mark, otherwise do what comes after the colon. This is a way of writing a short if/then statement.

Give the size of an object in memory, in bytes. The operand can be supplied inside parenthesis like a function call or without.

Example: the number 100 can be negative with the unary minus(-) sign like this: -100

Cast operator

Used to change one data type to another by placing the new data type in parenthesis directly before the current data type.
Example: int x = (int)'a'; assigns 97 to x. 97 is the integer value for the ascii letter a.
